System Pipeline Architecture

1. User Input
URL, File, Image, or Text entered via Dashboard.
2. Pre-Processing
Feature Extraction, Tokenization, & Resizing.
3. AI Inference
Data routed to specific ML/DL Model for analysis.
4. Risk Report
Score calculation & generation of actionable insights.

Machine Learning Model Specifications

Phishing Detector
Supervised Learning

Algorithm: Random Forest Classifier

Why this model? It builds multiple decision trees and merges them to get a more accurate and stable prediction, handling non-linear URL features effectively.

Key Role: Analyzes lexical features (URL length, special characters, domain age) to classify links as 'Legitimate' or 'Phishing'.

Deepfake Analyzer
Deep Learning

Algorithm: CNN (MesoNet / Xception)

Why this model? These specialized Convolutional Neural Networks look for microscopic artifacts and compression loss at the pixel level that human eyes miss.

Key Role: Detects face-swapping and frame-by-frame inconsistencies in synthetic video media.

Malware Scanner
Hybrid (ML + Static)

Algorithm: XGBoost + PE Header Analysis

Why this model? XGBoost is extremely fast for tabular data extracted from file headers (PE imports, section entropy) without needing to run the virus.

Key Role: Classifies executables as malicious based on structural metadata and entropy levels.

User Behavior (UEBA)
Unsupervised Learning

Algorithm: Isolation Forest

Why this model? It isolates anomalies (outliers) directly rather than profiling "normal" points, making it perfect for detecting hacked accounts.

Key Role: Flags impossible login travel times or unusual access hours that deviate from the user's history.

Network Analyzer
Deep Learning

Algorithm: LSTM Autoencoders

Why this model? Long Short-Term Memory (LSTM) networks understand sequences of time, allowing them to spot irregular patterns in data packet flow.

Key Role: Detects Zero-Day intrusions or DDoS attacks by identifying traffic that violates learned temporal patterns.

Email Forensics
Natural Language Processing

Algorithm: Multinomial Naive Bayes

Why this model? It calculates the probability of a text being "Spam" based on word frequency (TF-IDF), highly effective for text classification.

Key Role: Scans email bodies for urgency, financial keywords, and social engineering language.

Data Poisoning Monitor
Unsupervised Learning

Algorithm: DBSCAN Clustering

Why this model? Density-Based Spatial Clustering finds points that are closely packed and marks points in low-density regions as outliers (poisoned data).

Key Role: Protects ML training datasets by identifying malicious samples injected by attackers.

Fake Login Detector
Computer Vision

Algorithm: Siamese Neural Networks

Why this model? Uses "One-Shot Learning" to compare the visual similarity of a suspicious page against known legitimate login pages (e.g., Facebook, Google).

Key Role: Identifies UI cloning attacks even if the underlying code is obfuscated.

SQL Injection Tester
Supervised Learning

Algorithm: SVM (Support Vector Machine)

Why this model? SVMs are excellent at drawing a clear boundary between normal text strings and malicious SQL query structures.

Key Role: Analyzes input strings for SQL syntax patterns (e.g., ' OR 1=1) to prevent database breaches.

Password Analyzer
Information Theory

Algorithm: Shannon Entropy Calculation

Why this model? It measures the unpredictability of information content. Higher entropy means a password is harder to brute-force.

Key Role: Mathematically scores password complexity beyond simple regex rules.

Dark Web Checker
NLP / Retrieval

Algorithm: Named Entity Recognition (NER)

Why this model? NER (using models like BERT) scans unstructured text dumps to locate and classify Personally Identifiable Information (PII) like emails or SSNs.

Key Role: Monitors breach datasets for user credentials.

BugHunter Scanner
Static Analysis

Algorithm: Abstract Syntax Tree (AST) Parsing

Why this model? Converts code into a tree structure to understand logic flow, allowing detection of insecure function calls (e.g., eval(), os.system).

Key Role: Identifies security flaws in Python/C++ code snippets.

Text Encryptor
Cryptography

Algorithm: AES-256 (Advanced Encryption Standard)

Why this model? It is the industry standard for symmetric encryption, virtually unbreakable by brute force with current technology.

Key Role: Secures sensitive text data using military-grade encryption.

Metadata Extractor
Digital Forensics

Algorithm: EXIF Header Parsing

Why this model? Uses deterministic parsing to read the Exchangeable Image File Format standard embedded in media files.

Key Role: Reveals hidden GPS coordinates, camera models, and timestamps in images.
